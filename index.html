Stacks

Inventory Method: LIFO (Last In, First Out)


Common Operations:

  • Push: adds a new element to the top of the stack.

  • Pop: removes and returns the element on the top.

  • Size: returns the size of the stack.

  • Top/Peek: returns the element on the top without removing it.

  • Empty: returns a boolean value to indicate if the stack is empty.


Some Implementations:

  • Static: recomended when there's a maximum number of elements to be added to the stack.

  • Dynamic: recomended when unlimited elements can be added to the stack.

Queues

Inventory Method: FIFO (First In, First Out)


Common Operations:

  • Enqueue: adds a new element to the end of the queue.

  • Dequeue: removes and returns the firts element of the queue.

  • Size: returns the size of the queue.

  • Empty: returns a boolean value to indicate if the queue is empty.


Some Implementations:

  • Static: with a circular array. Recomended when there's a maximum number of elements to be added to the queue.

Lists

Description: lists are flexible data structures that can be implemented in many diferent ways.


Common Operations:

  • Insertion: adds a new element to the list.

  • Deletion: removes and returns an element from the list.

  • Size: returns the size of the queue.

  • Search: returns a boolean value to indicate if the the element passed as an argument is on the list.


Some Implementations:

  • Doubly Linked-List: a common implementation that consists of each node having links to its previous and next nodes.

  • Skip-List: an ordered structure that allows O(log n) complexity for search and insertion.

AVLs

Description: AVLs are self-balancing binary search trees, in other words, the height of the two child subtrees of any node differ by, at most, in one node (rebalancing is done to ensure this).


Common Operations:

  • Insertion: adds a new element to the tree.

  • Deletion: removes and returns an element from the tree.

  • Size: returns the size of the tree.

  • Search: returns a boolean value to indicate if the the element passed as an argument is on the tree.

Heaps

Description: heaps are complete and partially ordered binary trees.


Common Operations:

  • Insertion: adds a new element to the heap.

  • PopRoot: removes and returns the root element.

  • Root: returns the size of the queue.

  • Empty: returns a boolean value to indicate if the heap is empty.


Some Implementations:

  • Priority Queue: static implementation of a minHeap (the root has the smaller number on the heap)
